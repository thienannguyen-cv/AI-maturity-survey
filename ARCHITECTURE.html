1) Context & Decisions

  • Gate A: Approved.
  • Gate B: Approved.

2) Components (MVP v1)

Survey Source

Existing survey app on Vercel.

Ingest Endpoint

Serverless webhook receiver, validation + fingerprint.

Raw Store

raw_submissions keeps original payload for traceability.

Normalized Store

survey_responses, demographic_answers, question_answers.

Semantic SQL Views

Tier, maturity%, N/A rate, cohort deltas.

Dashboard

Metabase free reads views only.

Insight Job

Weekly summary with confidence tags.

Data Quality

Integrity/constraints/edge-case checks + alert hooks.

3) Data Flow

  1. User submits survey.
  2. Webhook validates schema, stores raw payload.
  3. Transformer writes normalized rows.
  4. Metric job computes semantic views.
  5. Dashboard queries only stable semantic views.
  6. Weekly insight report filters cohorts with n >= 20.

5) Free Deployment Steps

  1. Create D1 database and apply schema SQL.
  2. Deploy ingest worker and connect survey webhook target.
  3. Run seed/mock loader to verify dashboard tab wiring.
  4. Deploy Metabase free and connect read-only DB user.
  5. Import dashboard templates and validate each tab.
  6. Enable scheduled data quality checks and basic alerts.

6) Tests & Data Checks (Required)

  • Integrity: non-null keys, FK references, uniqueness constraints.
  • Range checks: score levels restricted to 1..5 or N/A.
  • Dedup checks: repeated session bursts handled by fingerprint + timestamp rules.
  • Metric checks: maturity% consistent with SQL contract.
  • Edge cases: all N/A response, partial completion, mixed language text, malformed payload.

7) Change Protocol

Any requested change should update: (1) semantic SQL, (2) this HTML section, and (3) smoke-check evidence.
