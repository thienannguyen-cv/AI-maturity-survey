AI Maturity Community Dashboard — Architecture & Delivery Plan

Tài liệu này chốt kiến trúc 100% free ở giai đoạn đầu, tối ưu tốc độ triển khai và minh bạch để bạn iterate nhanh.

1) Mục tiêu & phạm vi

  • MVP trong 5 ngày: ingest dữ liệu survey, tính metrics chuẩn, hiển thị dashboard cộng đồng.
  • Tách data layer khỏi UI layer để chuyển đổi Metabase ↔ Static SQL+JS dễ dàng.
  • AI thực hiện phần kỹ thuật, bạn đóng vai trò phê duyệt/tinh chỉnh.

2) Kiến trúc thành phần

  1. Survey Frontend (đang chạy Vercel): gửi payload qua webhook.
  2. Ingest Worker (Cloudflare Workers free): validate + chuẩn hóa + dedupe.
  3. Data Store (Cloudflare D1 free): raw + normalized + semantic views.
  4. Metrics Job (cron): refresh views/materialized tables mỗi 15 phút.
  5. Dashboard v1 (Metabase free): đọc trực tiếp từ semantic views.
  6. Mirror Dashboard (HTML mock tab): phản ánh UI/luồng để bạn review nhanh trước khi chỉnh dashboard thật.

3) Luồng dữ liệu

Survey Submit -> Webhook -> Ingest Worker -> raw_submissions
                                   |-> normalized tables
                                   |-> quality_checks
normalized -> semantic views -> Metabase dashboard
semantic + checks -> weekly insight summary

5) Metric contract (đã duyệt)

maturity_percent = total_weighted_score / applicable_max * 100
applicable_max = SUM(max_weighted_score WHERE is_na = false)
cohort threshold = 20
confidence tags = High / Medium / Low

6) Dashboard tabs (v1)

  • Overview: sample size, tier distribution, radar by capability group.
  • Cohort Explorer: so sánh role/team-size/time-with-AI/risk domain.
  • Question Diagnostics: low-score items, variance, N/A rate.
  • Trend & Stability: biến động theo tuần + composition shift.
  • Feedback Intelligence: topic cluster + anonymized excerpts.
  • Change Request Workspace: tab riêng để bạn ghi chú chỉnh sửa có cấu trúc.

8) Kế hoạch triển khai chi tiết

  1. Ngày 1-2: ingest worker + schema + dedupe + audit log.
  2. Ngày 3: metric SQL views + data quality checks.
  3. Ngày 4: dashboard v1 + filters + cohort threshold 20.
  4. Ngày 5: smoke tests + runbook deploy + handover.

9) Testing checklist (thực chất)

  • Integrity: missing fields, invalid question IDs, out-of-range levels.
  • Constraints: duplicate session burst, timestamp ordering, version compatibility.
  • Edge cases: all N/A responses, partial completion, single cohort dominance.
  • Metric correctness: recompute manually on fixture samples.

10) Tiêu chí nghiệm thu

PASS khi dashboard cập nhật trong 15-30 phút, metric truy vết được về SQL/view, và bạn có thể gửi change request qua tab workspace để AI đồng bộ lại dashboard thực tế.

Lưu ý: trước khi có dữ liệu lớn, insight chỉ ở mức directional, không suy diễn nhân quả.
